实现落点

统一恢复入口

ncclCommPrepareSurvivorTopo() 在 parent rank 空间内连接 survivor Ring 邻居；ncclCommActivateSurvivorTopo() 再切换旧 communicator 的 Ring 执行视图。

双 rank 空间

transport 邻接仍使用 parent rank；设备执行视图更新为 survivor index，包括 devComm->nRanks、devComm->rank 和 ring.userRanks。

Root 参数转换

实验程序对 Broadcast 与 Reduce 将用户指定的 parent root 转为 survivor index 后调用原生 API，验证任意存活 root 的 Ring 语义。

实验驱动范围

run_collective_matrix.sh 当前只接受 --algo ring，并用单个 SIGKILL 故障覆盖五种集合通信及 root 组合。

验证进展

Ring 基线 解决 ring.userRanks 与 device-side rank 映射后，五个原生 Ring collective 的 kill/root 边界矩阵通过。
当前删减 相比含 Tree 实验的中间工作版本，当前已删除 Tree transport 连接、Tree device 状态发布以及新增 Broadcast/Reduce Tree kernel。
验证依据 2026-05-26 的 Ring 矩阵在 np=4 下执行 36 个 case：四个失败 rank、五种 collective，以及 Broadcast/Reduce 的全部存活 root 组合均通过。

仍需说明的边界

  • 实验验证模型是 parent communicator 中单个 rank 被终止后，由全部 survivor 继续执行；API 对任意 survivor 列表的接受不等于相关组合已通过验证。
  • 当前明确验证基线为 NCCL_PROTO=SIMPLE，未将 LL / LL128 纳入结论。
  • Survivor 恢复仅对外主张 Ring；任何 Tree、PAT、NVLS 或 CollNet 路径均不属于当前能力。
  • PASS_FORCE_CLEANUP 表示 survivor 已正确完成后，脚本主动清理仍阻塞的 launcher 进程组。
